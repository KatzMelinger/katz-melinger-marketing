# 2.2 — Dedicated Social Copy Generator (scope)

## Problem
Social copy is generated by the **blog** system. Both the Repurpose "Generate posts"
action (`app/api/content-production/social/route.ts:58`) and the repurpose-variations flow
(`app/api/content-production/repurpose/route.ts`) call `generateMultiFormat()` — the same
function blog uses — with **soft** per-platform length guidance in `FORMAT_INSTRUCTIONS`
(`lib/content-multiformat.ts:54`). Nothing enforces a cap, so output reads like blog chunked
into slides (Diana's 8-slide carousel with full paragraphs). No hook formulas, no one-angle
rule, no source tag, no quality checklist.

The recent social work (calendar management, Ayrshare scheduling, carousel **image**
rendering, the six ops screens) is all **distribution/ops** — it does not touch generation.

## Goal
A dedicated social generator, separate from blog, that enforces the social rulebook
(Rules 1–10) at generation time: hard length caps, hook formulas, three-part structure,
source-required, sensitive-topic tone, quality checklist on the card. Keep the existing
scheduling / carousel-image / ops layer unchanged.

## What we keep (no rebuild)
- Scheduling + distribution: `lib/ayrshare.ts`, `social/route.ts` Mon/Wed/Fri plan, `social_posts`.
- Carousel **images**: `lib/carousel-slides.ts` (parser) + `lib/carousel-render.ts` (PNG per slide).
- Ops screens: Calendar, KPI, Trends, Best Time, Hashtags, Competitors.
- Cross-cutting fixes already in: `lib/sanitize-content.ts` (Rule 6 dashes),
  `lib/sensitive-topic.ts` (Rule 7 tone) — both already fire on the social path.
- `lib/content-dedup.ts` (for Rule 8 duplicate-angle), `firm-context`, `content-skills`.

## What we build

### New: `lib/social-format-rules.ts` (data)
Per-format hard caps + hook formulas + banned openers, as a typed table:
- CAROUSEL: 5–7 slides (max 8), hook slide <12 words, body slide 15–25 words, CTA slide <15 words.
- LINKEDIN: 50–150 words. FACEBOOK: 50–120. INSTAGRAM caption: hook + 2–4 lines, hashtags separate.
- TWITTER single <280 chars; THREAD 4–6 tweets, each <280. TIKTOK/REELS 75–110 words (30–45s).
- Hook formulas (question / scenario / myth-bust / stat / direct-address); banned openers list.

### New: `lib/content-social.ts` (generator)
`generateSocialPosts({ source, formats, tenantId })` →
- **Dedicated** system + user prompt built from the rules table (NOT `generateMultiFormat`).
- **Rule 1 source-required**: takes an approved source (published blog / case result / legal
  update / service page). No blank-topic generation.
- **Rule 2** one-angle extraction; **Rule 3/4** hook + hook→value→soft-CTA structure.
- **Rule 5 enforcement**: validate each output against caps. On overflow → one regen with a
  tighter instruction; if still over, hard-trim (carousel: drop to 7 slides / clamp slide words).
- Runs `stripEmDashes` (Rule 6) + `sensitiveToneBlock` (Rule 7) — reuse.
- Returns `{ body, format, checklist }` where `checklist` = Rule 10 pass/fail booleans.
- Saves to `content_drafts` with `metadata.social_source` (the tagged source) +
  `metadata.social_checklist`.

### Edits
- `app/api/content-production/social/route.ts` — swap `generateMultiFormat` → `generateSocialPosts`.
- `app/api/content-production/repurpose/route.ts` — same swap; source = the page being repurposed.
- `components/repurpose-review-drawer.tsx` — render source tag + per-post checklist (mirror the
  blog QA checklist pattern in `components/draft-drawer.tsx:280-302, 972+`).
- `app/social/content-calendar/page.tsx` `PostDetailDrawer` — show source asset + checklist.
- `lib/content-multiformat.ts` — leave as-is for blog/email; social/carousel/video_short simply
  stop routing through it. (`FORMAT_INSTRUCTIONS` social entries become dead for the social path.)

### Rule 8 (duplicate-angle)
Before finalizing, compare the chosen source+angle against this month's `social_posts`
(scheduled/published). Reuse `content-dedup` semantic keys or a light angle key. On collision,
pick a different angle from the same source or flag to Diana.

## Phasing (shippable incrementally)
- **Phase 1 (fixes Diana's actual complaint):** dedicated generator + hard caps (Rule 5) +
  hook/structure (Rules 3–4) + reuse dashes/tone. Swap the two entry points. → output stops
  being oversized blog-in-slides.
- **Phase 2:** source-required + source tag on card (Rule 1), quality checklist on card (Rule 10).
- **Phase 3:** duplicate-angle check (Rule 8), regenerate-on-fail loop, one-angle refinement.

## Decisions (locked 2026-07-07)
1. **Carousel slide count** — 5–7 slides, hard max 8. Enforced.
2. **Checklist** — caps enforced at generation (hard); card checklist is **advisory** (shows
   pass/fail, does not block scheduling), like the blog's soft SEO checks.
3. **Source-required** — **block** source-less AI generation. `generateSocialPosts` throws when
   no approved source is supplied; source-less posts stay manual Content Studio entries.
4. **TikTok/Reels** — **include now** (maps to the existing `video_short` format; Craig LeVay
   script rules: 75–110 words, 30–45s, dual-audience, no hype/guarantees).

## Effort
Phase 1 ≈ 2–3 days. Full (all three phases) ≈ 1.5–2 weeks.
